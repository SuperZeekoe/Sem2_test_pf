"Crap" to-do list

Next assignment that was more of a bigger JavaScript assignment, was the “Crap-to-do-list”. The objective was to create a functioning to-do list with JavaScript, no matter what the code or styling looks like. Because Frank called it a “Crap” to-do list, I chose to go with a slightly juvenile approach of styling it after “poop”.

I had a few initial versions that worked slightly, but with the help of Rob it came together quite quickly as he’s pretty good at this stuff. It seems I have the tendency to overthink these coding challenges and make things too hard for myself initially, as my first thought went to using JSON for the list. That was a bit too much which I spent too much time on unnecessarily.

In the end it simply creates a 'li' element underneath the chosen #id after an Event Listener. The 'li' element contains the data entered in another text box which gets read out after clicking the same event listener.

Portoflio Coding

My portfolio has been a thorn in my side every since the first evaluation. Once again I shot myself in my foot, trying to create something innovative and interesting compared to the standard layouts.

Following the initial tutorial for the click & drag navigation on the homepage (which didn’t even work and I had to tweak some things, thanks HyperPlexed), I wanted to make it a one-pager, having only vertical movement upon clicking certain elements.
